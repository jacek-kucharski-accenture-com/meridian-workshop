Known Gaps (baseline, April 2026)

R1 — Reports filter disconnect

Reports.vue does not subscribe to useFilters. The global FilterBar has no effect on the Reports page. Backend reports endpoints also accept no filter query params.

R1 — Reports i18n missing

All labels in Reports.vue are hardcoded English strings. The t() translation function and currentCurrency from useI18n are not wired up.

R2 — No restocking endpoint

No /api/restocking endpoint exists. inventory.json and demand_forecasts.json have the data needed to compute recommendations; endpoint and Vue view are to be built.

R3 — No browser tests

No Playwright or other E2E test suite exists. Backend unit tests exist in tests/backend/ but no front-end or integration coverage.

Options API remnants

Previous vendor left some components using the Vue 2-style Options API. Migration to Composition API is incomplete across the codebase.

/api/tasks not implemented

api.js has getTasks(), createTask(), deleteTask(), toggleTask() methods but main.py has no matching routes. These calls will 404 at runtime.
